HyperFx is a real-time bridge that captures LED color data from **Hyperion-NG** (ambilight) and forwards it to **LedFx** virtual devices for synchronized ambient lighting. It continuously reads Hyperion-NG's LED color stream over WebSocket, applies configurable color processing (sampling, averaging, brightness enhancement), and pushes the result as an Energy effect to LedFx via REST API.

```
Screen Content  →  Hyperion-NG  →  WebSocket  →  HyperFx  →  REST API  →  LedFx  →  LED Strips
```

## Features

- **Real-time sync** — continuous capture-and-forward loop with sub-second latency
- **Configurable sampling** — adjustable sample count and refresh rate for responsiveness vs. smoothness
- **Smart color processing** — weighted averaging, dark-color enhancement, minimum brightness floor
- **Resilient fallback chain** — WebSocket primary → REST fallback → graceful skip
- **Health monitoring** — periodic connectivity checks with sync/error counters
- **Single-file deployment** — one Python script, pip dependencies, no complex setup
